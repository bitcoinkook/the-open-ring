What this is

The Signal Ledger is the Field node of the Open Ring cycle. It records what happened when someone ran the diagnostic and acted on the verdict — what held, what didn't, what the real world showed.

A ledger only means something when nobody owns it. Records here are signed Nostr events — owned by the person who filed them, verifiable by anyone, stored across the relay network.

The framework holds itself to its own condition: α × P(detect) > 1. A Google Sheet fails both. A signed Nostr event satisfies both.

Event schema — kind 1337
kind: 1337 content: { full field record as JSON } tags: [ ["t", "signal-ledger" ], ["t", "open-ring" ], ["structure", "..." ], ["state", "INVERTED|..." ], ["break", "MEDIUM|..." ], ["inversion_type", "accretion|..." ], ["action_taken", "yes|no|pending" ], ["client", "open-ring-signal-ledger"] ]

Any client that understands this schema can read and write to the ledger. The schema is MIT licensed. Build on it.

How to connect

The Signal Ledger uses NIP-07 — a browser extension that holds your Nostr private key and signs events without ever exposing the key to this page.

Compatible extensions: Alby, nos2x, Nostore. Install any one, import or generate a key, then click Connect.
