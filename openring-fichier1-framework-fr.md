*Ceci est la version portable d'Open Ring — un diagnostic qui fonctionne dans n'importe quel chat IA (Grok, ChatGPT, Claude) et produit un output conforme au stack du Protocole Open Ring (ORP-01 / ORP-02 / ORP-03).*

---

## QU'EST-CE QU'OPEN RING

Open Ring identifie les inversions structurelles — des situations où les mauvaises choses sont fixes et les mauvaises choses varient, aux dépens de l'utilisateur. Il produit un verdict complet : ce qui ne va pas, pourquoi ça persiste, quelle est l'intervention minimale, et quoi faire.

La même logique s'applique à chaque domaine. Marchés, institutions, relations, produits, systèmes, pratiques. La surface change. La condition sous-jacente non.

---

## LA CONDITION CENTRALE

```
I(B) est définie  ⟺  α × P(détecter) > 1
```

Les informations sur tout élément variable B n'existent que lorsqu'une référence constante A est genuinement stable.

A est genuinement constante seulement quand la falsifier est à la fois coûteux (α) et détectable de manière fiable (P(détecter)). Si l'un des deux fait défaut, A n'est que nominalement constante — elle semble fixe mais dérive.

Quand cela se produit, le signaler explicitement : **A N'EST QUE NOMINALEMENT CONSTANTE.**

En langage simple : une chose ne te dit la vérité que lorsque mentir à son sujet coûte plus cher que dire la vérité.

## LES CINQ ÉTATS

```
A=C, B=V  →  SAIN           La référence tient. Les informations circulent.
A=C, B=C  →  GELÉ           La référence tient mais rien ne bouge.
A=V, B=V  →  SANS ANCRAGE   Tout dérive. Pas de référence.
A=V, B=C  →  INVERSÉ        Capture. L'utilisateur paie. Le titulaire gagne.
—  —  —   →  IMMUNE         Ne peut pas être capturé. État sain terminal.
```

Le pattern de capture est toujours INVERSÉ. Le coût retombe toujours sur l'utilisateur. La capture ne nécessite pas de mauvais acteurs — seulement des incitations mal alignées.

**Capture par accrétion :** personne ne l'a conçue. L'infrastructure, l'habitude et l'inertie se sont accumulées jusqu'à ce que la mauvaise chose se fixe.

**Capture intentionnelle :** un titulaire a activement conçu ou défend la fixité.

Les deux produisent le même résultat pour l'utilisateur. Nomme laquelle est présente — la bonne intervention en dépend.

## LES CINQ STRATES

Chaque structure existe simultanément sur cinq niveaux. Une inversion peut être visible dans une strate et maintenue dans une autre. Le cristal graine doit aller là où se trouve le blocage, pas nécessairement là où l'inversion apparaît.

```
1. INDIVIDUEL      — Conscience / Capacité / Énergie / Identité
2. RELATION        — Transparence / Confiance / Réciprocité / Statut
3. INSTITUTION     — Éducation / Distribution / Finance / Autorité
4. ENVIRONNEMENT   — Visibilité / Infrastructure / Biens communs / Régulation
5. CULTURE         — Récit / Langage / Valeur / Sens
```

Vérifie toujours les cinq. L'écart entre où l'inversion apparaît et où elle est maintenue est là où vit la vraie intuition.

## LES QUATRE COUCHES DIAGNOSTIQUES

Chaque couche correspond à une section d'ORP-01. Le fichier opérateur les parcourt dans l'ordre.

### Couche 1 — Diagnostic d'Inversion (ORP-01 §2)

P2.01. Qu'est-ce qui est fixe ? Nomme-le avec précision.
P2.02. Qu'est-ce qui varie ? Nomme-le avec précision.
P2.03. Qui supporte le coût ? Utilisateur / titulaire / les deux. Nomme le coût spécifiquement — argent, temps, santé, opportunité, attention, autonomie, sécurité.
P2.04. Quelle est l'importance du coût ? Grave / significatif / modéré / mineur.
P2.05. Type de capture — intentionnelle, accrétion, mixte, ou aucune.
P2.06. Vérification coût-détection. α est-il élevé ? P(détecter) est-il élevé ? Si l'un est faible, la référence n'est que nominalement constante.

### Couche 2 — Gradient (ORP-01 §3)

P3.01. Pression naturelle — en quoi cette structure veut-elle se transformer si elle est débloquée ?
P3.02. Énergie de maintien — qu'est-ce qui est dépensé pour maintenir le blocage en place ?
P3.03. Point de supersaturation — où la pression s'est-elle accumulée le plus longtemps sans relâchement ? C'est là que l'appel à l'action atterrit.
P3.04. Durée de la pression — décennies, années, mois, récent.
P3.05. Cristal graine — l'intervention minimale qui permet à la structure correcte d'émerger d'elle-même. Placé dans la strate où le blocage est maintenu.

### Couche 3 — ChaîneDeSignal (ORP-01 §4)

Quatre nœuds en séquence. Arrête-toi au premier défaut. Le cristal graine appartient au point de défaut.

```
P4.01  SIGNAL      Des informations genuines parviennent-elles aux personnes qui en ont besoin ?
P4.02  MOYEN       Existe-t-il un canal de transmission non capturé par le titulaire ?
P4.03  POTENTIEL   Est-ce que α × P(détecter) > 1 ?
P4.04  PHASE       Est-ce seulement local, ou distribué globalement ?
P4.05  POINT DE DÉFAUT — premier nœud défaillant. Plus la strate où le défaut se produit.
```

### Couche 4 — Orientation (ORP-01 §5)

P5.01. Invariant correct — qu'est-ce qui devrait être définitivement fixe, comme orientation, pas comme valeur ?
P5.02. Ce qui doit être libéré — qu'est-ce qui doit varier que la structure actuelle bloque ?
P5.03. Pilier le plus faible — signal, moyen, potentiel ou phase.
P5.04a. Vérifications d'immunité — quatre questions binaires :
- L'éducation sur cette structure peut-elle être supprimée par un seul acteur ?
- La distribution nécessite-t-elle un canal contrôlé ?
- Un titulaire peut-il concentrer des financements pour la bloquer ?
- La légitimité dépend-elle d'une validation externe ?
- Les quatre Non → approchant l'immunité à la capture.

P5.04b. Le plus proche d'être satisfait — éducation / distribution / finance / légitimité / les quatre / aucune.
P5.04c. Prochaine étape vers l'immunité.

## LE VERDICT À NEUF CHAMPS

L'opérateur produit un verdict conforme à ORP-02 avec neuf champs dans cet ordre. Les neuf sont obligatoires.

```
V.01  STRUCTURE               ce qui a été analysé
V.02  ÉTAT                    SAIN / GELÉ / SANS ANCRAGE / INVERSÉ / IMMUNE
V.03  TYPE D'INVERSION        accrétion / intentionnelle / mixte / aucune
V.04  POIDS DU SIGNAL         fort / adéquat / faible
V.05  POINT DE DÉFAUT         SIGNAL / MOYEN / POTENTIEL / PHASE / aucun
V.06  DIAGNOSTIC              une phrase en langage simple
V.07  TRAITEMENT              le cristal graine, placé correctement
V.08  APPEL À L'ACTION        un mouvement spécifique — qui, quoi, quand
V.09  CHEMIN D'IMMUNITÉ       jusqu'où de l'immunité, quoi ensuite
```

## LES TROIS DRAPEAUX

Obligatoires dans le verdict quand leurs conditions sont remplies :

```
DRAPEAU A  NOMINALEMENT CONSTANT    si P2.06 α = faible OU P(détecter) = faible
DRAPEAU B  PRÉLIMINAIRE             si P1.04 = de seconde main ou spéculatif
DRAPEAU C  DÉCALAGE DE STRATE       si strate P3.05 ≠ strate P4.05 de défaut
```

## LE CRISTAL GRAINE

L'output le plus important de tout diagnostic Open Ring.

Un cristal graine n'est pas une solution, un produit ou une campagne. C'est la chose minimale vraie placée dans la bonne strate qui permet à la structure de se réorganiser correctement d'elle-même.

Propriétés :
- Minimal — la plus petite intervention possible qui lance le processus
- Placé dans la bonne strate — là où le blocage est maintenu, pas là où il apparaît
- Vrai — vérifiable factuellement, pas une affirmation ou une opinion
- Ouvert — disponible pour tous, sans propriétaire
- Permet l'émergence — la structure correcte grandit à partir de lui plutôt que d'être imposée

Un cristal graine a fait son travail quand la structure correcte se propage sans autre intervention de la part de celui qui l'a placé.

---

## CE QUE PRODUIT UN SIGNAL FAIBLE

Le framework organise ce que l'utilisateur sait déjà. Il ne remplace pas le savoir.

L'expérience directe avec une structure produit un verdict vrai et actionnable. La connaissance de seconde main de la même structure produit un output qui sonne similaire mais est creux en dessous. Si le signal est faible, le diagnostic le signale et continue — mais marque le verdict comme PRÉLIMINAIRE.

## LE STACK DE PROTOCOLES

La version pocket d'Open Ring exécute le Protocole Open Ring dans une fenêtre de chat.

```
ORP-01   Protocole de Questions    les questions que cet opérateur pose
ORP-02   Protocole de Verdict      le format d'output à neuf champs
ORP-03   Protocole de Terrain      le registre signé sur Nostr kind 1337
                                   — déposé dans le Signal Ledger après avoir agi
```

L'opérateur couvre ORP-01 et ORP-02 dans le chat. L'utilisateur apporte le verdict au Signal Ledger pour compléter ORP-03.

```
Signal Ledger : bitcoinkook.github.io/the-open-ring/signal-ledger-nostr.html
```
